# S-Art

S-Art — учебное полнофункциональное веб-приложение в формате социальной сети
для авторов визуальных работ и пользователей генеративного ИИ.

В приложении можно публиковать изображения, хранить промпты, просматривать
ленту, добавлять записи в избранное, подписываться на авторов через систему
друзей и управлять содержимым с помощью роли администратора.

## Возможности

### Пользователи

- регистрация и вход по email и паролю;
- JWT-авторизация с сохранением сеанса в браузере;
- просмотр собственного и чужих профилей;
- изменение имени, фамилии, статуса и описания;
- загрузка аватара;
- светлая и тёмная тема оформления.

### Работы

- общая лента изображений;
- фильтрация по рубрикам;
- создание работы с изображением, названием и коротким описанием;
- редактирование названия, описания и рубрики;
- удаление собственной работы;
- увеличение изображения по щелчку;
- повторный щелчок или закрытие окна возвращает обычный размер;
- добавление работы в избранное с помощью значка сердца.

Поддерживаемые форматы: JPG, JPEG, PNG, GIF, WebP и AVIF. Максимальный размер
загружаемого файла — 10 МБ.

### Рубрики

- Иллюстрации;
- Персонажи;
- Городские сюжеты;
- Природа и пейзажи;
- Фэнтези;
- Портреты;
- Архитектура и интерьеры;
- Абстракция и эксперименты;
- Прочее.

### Промпты

- создание и хранение текстовых запросов для генеративного ИИ;
- указание названия и модели;
- публикация или хранение непубличного промпта;
- редактирование и удаление собственных промптов;
- копирование публичного промпта в собственную коллекцию;
- добавление промптов в избранное.

### Друзья

- отправка запроса другому пользователю;
- отображение состояния «Запрос на добавление»;
- счётчик входящих запросов в шапке;
- принятие и отклонение запросов;
- переход из списка друзей в профиль пользователя;
- удаление пользователя из друзей.

### Интерфейс

- фиксированная верхняя панель навигации;
- фиксированный футер со статистикой пользователей, постов и промптов;
- встроенный справочник по клавише `F1`;
- закрытие справочника клавишей `Esc`;
- адаптивное карточное представление.

## Технологии

### Frontend

- Vue 3;
- Vue Router;
- Pinia;
- Axios;
- Vite 5;
- JavaScript и CSS.

### Backend

- Python;
- FastAPI;

## Быстрый запуск

### Windows

Откройте папку проекта и запустите:

```powershell
.\start.bat
```

Сценарий:

1. установит зависимости backend;
2. установит зависимости frontend;
3. откроет backend на `http://localhost:8000`;
4. откроет frontend на `http://localhost:5173`.

Окна backend и frontend должны оставаться открытыми во время работы.

### Linux и macOS

```bash
chmod +x start.sh
./start.sh
```

Остановка выполняется сочетанием `Ctrl+C`.

## Ручной запуск

Ручной способ удобнее для разработки и диагностики. Backend и frontend нужно
запускать в двух отдельных терминалах.

### 1. Backend

Windows PowerShell:

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
$env:JWT_SECRET="замените-на-длинный-случайный-секрет"
python main.py
```

Linux/macOS:

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
export JWT_SECRET="замените-на-длинный-случайный-секрет"
python main.py
```

Проверка backend:

```text
http://localhost:8000/api/health
```

Ожидаемый ответ:

```json
{"status":"ok"}
```

Интерактивная документация API:

```text
http://localhost:8000/docs
```

### 2. Frontend

Во втором терминале:

```powershell
cd frontend
npm install
npm run dev
```

Откройте:

```text
http://localhost:5173
```

Vite автоматически перенаправляет запросы `/api` и `/uploads` на backend,
работающий на порту `8000`.

## Работа с приложением

### Создание работы

1. Войдите в аккаунт.
2. Нажмите на аватар в шапке.
3. В блоке «Добавить своё» нажмите `+ Работа`.
4. Заполните название и короткое описание.
5. Выберите рубрику.
6. Выберите изображение размером до 10 МБ.
7. Нажмите «Сохранить».

Работа появится в профиле и общей ленте. На собственной карточке доступны
команды «Изменить» и «Удалить». При редактировании изображение остаётся прежним.

### Создание промпта

1. Откройте собственный профиль.
2. Нажмите `+ Промпт`.
3. Укажите название, текст запроса и модель.
4. При необходимости измените признак публичности.
5. Нажмите «Сохранить».

### Избранное

Нажмите на сердце в карточке. Заполненное розовое сердце означает, что запись
сохранена. Повторное нажатие удаляет запись из избранного.

### Увеличение изображения

Щёлкните по изображению в карточке. Оно откроется в увеличенном режиме.
Повторный щелчок сворачивает изображение.

### Темы

Переключатель в шапке меняет светлую и тёмную тему. Выбор сохраняется в
`localStorage` браузера.

### Справочник

Нажмите `F1` в любой основной части приложения или кнопку «F1 Справочник» в
футере. Для закрытия используйте `F1`, `Esc` или кнопку `×`.

## Роли и администрирование

В системе предусмотрены две роли:

- `user` — обычный пользователь;
- `admin` — администратор.

Администратору доступен отдельный пункт навигации «Администратор». Возможности:

- просмотр пользователей;
- назначение и снятие административной роли;
- удаление пользователей вместе с их связями и контентом;
- просмотр всех работ и промптов;
- удаление любой работы или промпта.

Ограничения безопасности интерфейса:

- администратор не может удалить собственный аккаунт;
- администратор не может снять роль администратора с самого себя;
- backend повторно проверяет роль для каждого административного запроса.

## Демо-данные

Для повторного заполнения базы выполните:

```powershell
cd backend
python seed.py
```

Сценарий является идемпотентным: повторный запуск обновляет известные записи,
не создавая их копии. Он создаёт или обновляет демо-пользователей, работы,
промпты, избранное и несколько подтверждённых дружеских связей.

Особенности:

- пароль демо-пользователей — `demo123`;
- контент пользователя `q@q.com` дополняется, только если этот пользователь уже
  существует в базе;
- локальные изображения находятся в `backend/uploads/demo/q`;
- расположенные рядом TXT-файлы содержат промпты в UTF-8;
- часть фотографий загружается в интерфейс по открытым ссылкам Pexels.

Не удаляйте `backend/uploads`, если хотите сохранить изображения существующих
записей.

## API

Основной префикс — `/api`. Полная схема доступна в Swagger UI:
`http://localhost:8000/docs`.

| Метод и путь | Назначение | Авторизация |
|---|---|---|
| `GET /api/health` | Проверка backend | Нет |
| `POST /api/auth/register` | Регистрация | Нет |
| `POST /api/auth/login` | Вход | Нет |
| `GET /api/auth/me` | Текущий пользователь | Да |
| `GET /api/users/{id}` | Профиль пользователя | Да |
| `PUT /api/users/me` | Изменение профиля | Да |
| `POST /api/users/me/avatar` | Загрузка аватара | Да |
| `GET /api/posts` | Список работ | Нет |
| `POST /api/posts` | Создание работы | Да |
| `PUT /api/posts/{id}` | Изменение собственной работы | Да |
| `DELETE /api/posts/{id}` | Удаление собственной работы | Да |
| `GET /api/prompts` | Список промптов | Нет |
| `POST /api/prompts` | Создание промпта | Да |
| `POST /api/prompts/{id}/copy` | Копирование промпта | Да |
| `PUT /api/prompts/{id}` | Изменение собственного промпта | Да |
| `DELETE /api/prompts/{id}` | Удаление собственного промпта | Да |
| `GET /api/favorites` | Избранное пользователя | Да |
| `POST /api/favorites` | Добавление в избранное | Да |
| `DELETE /api/favorites/{id}` | Удаление из избранного | Да |
| `GET /api/friends` | Список друзей | Да |
| `GET /api/friends/requests` | Входящие запросы | Да |
| `POST /api/friends/{user_id}` | Отправка запроса | Да |
| `PUT /api/friends/{user_id}` | Принятие запроса | Да |
| `DELETE /api/friends/{user_id}` | Удаление связи/запроса | Да |
| `GET /api/stats` | Статистика футера | Нет |
| `/api/admin/*` | Управление системой | Администратор |

Защищённые запросы используют заголовок:

```http
Authorization: Bearer <JWT>
```

Frontend получает токен при входе, сохраняет его в `localStorage` и
автоматически добавляет к запросам.

## База данных и файлы

SQLite хранится в:

```text
backend/database.db
```

При старте backend:

- создаются отсутствующие таблицы;
- выполняется небольшая встроенная миграция поля `users.role`;
- существующие записи не удаляются.

Пользовательские изображения сохраняются в:

```text
backend/uploads/posts/
```

Backend публикует каталог `uploads` по URL `/uploads/...`.

### Резервное копирование

Для полного резервного копирования остановите backend и сохраните:

```text
backend/database.db
backend/uploads/
```

Одного файла базы недостаточно: изображения хранятся отдельно на диске.

```powershell
cd frontend

```

## Диагностика проблем

### `http://localhost:5173` не открывается

Проверьте:

```powershell
cd frontend
npm install
npm run dev
```

В терминале должна появиться строка с адресом Vite. Если порт занят, завершите
старый процесс или измените порт одновременно в `vite.config.js` и настройках
CORS backend.

### Интерфейс открыт, но регистрация или вход завершаются ошибкой

Чаще всего не запущен backend. Проверьте:

```text
http://localhost:8000/api/health
```

Если адрес не отвечает:

```powershell
cd backend
python -m pip install -r requirements.txt
python main.py
```

### Изображения не отображаются

- убедитесь, что backend работает;
- проверьте наличие файла в `backend/uploads`;
- не переносите только `database.db` без каталога `uploads`;
- для удалённых фотографий проверьте подключение к интернету;
- обновите страницу с помощью `Ctrl+F5`.

### Ошибка CORS

Backend разрешает запросы от `http://localhost:5173`. Если frontend запущен на
другом порту или хосте, обновите `allow_origins` в `backend/main.py`.

### Порт занят

Windows:

```powershell
Get-NetTCPConnection -LocalPort 8000,5173 -ErrorAction SilentlyContinue
```

Завершите ненужный процесс или измените настройки портов.

### Изменения демо-данных не появились

Повторите заполнение и перезапустите backend:

```powershell
cd backend
python seed.py
python main.py
```

После этого выполните `Ctrl+F5` в браузере.

### Повреждён локальный сеанс

Удалите ключ `token` из `localStorage` браузера или очистите данные сайта для
`localhost:5173`, затем войдите заново.

## Безопасность и ограничения

Проект предназначен для локальной разработки и демонстрации.

Перед публикацией в интернете необходимо:

- установить уникальное значение переменной окружения `JWT_SECRET`;
- ограничить CORS реальным доменом;
- убрать тестовые пароли и аккаунты;
- использовать production ASGI-сервер и HTTPS;
- добавить миграции Alembic;
- проверить MIME-тип и содержимое загружаемых файлов, а не только расширение;
- добавить ограничения частоты запросов;
- хранить изображения в управляемом файловом или объектном хранилище;
- настроить журналирование и резервное копирование;
- не публиковать `database.db` с реальными пользовательскими данными.

Текущая база SQLite и локальное файловое хранилище подходят для демонстрации,
но не рассчитаны на высокую нагрузку и одновременную работу большого числа
пользователей.
